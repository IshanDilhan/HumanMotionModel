========================================================================
HUMAN MOTION DETECTION PROCESS & LSTM MODEL DETAILS
========================================================================

------------------------------------------------------------------------
SECTION 1: THE EXACT PROCESS BEHIND DETECTION
------------------------------------------------------------------------
Every frame of the input source (webcam, video file, or camera) goes through 
the following 5 consecutive stages in real time:

1. FRAME CAPTURE & COLOR CONVERSION:
   - OpenCV (cv2.VideoCapture) reads a raw frame from the source.
   - The image is converted from BGR (OpenCV's standard format) to RGB 
     (required by the MediaPipe engine).

2. SKELETAL LANDMARK EXTRACTION:
   - The RGB image is processed by MediaPipe Pose (BlazePose).
   - MediaPipe detects a human body and returns 33 skeletal landmarks.
   - If a person is detected, the 3D (x, y, z) coordinates of all 33 points 
     are stored in a sliding window buffer (a queue with max length 30).
   - If no person is detected, the buffer is cleared and the output defaults 
     to "Stationary".

3. VELOCITY COMPUTATION & FEATURE SCALING:
   - Once the buffer contains at least 2 frames of keypoints, the code 
     calculates frame-to-frame coordinate differences using NumPy (np.diff):
         Velocity = Coordinates(t) - Coordinates(t-1)
     This converts absolute position values into velocity vectors.
   - FEATURE SCALING: The raw velocities are multiplied by a factor of 100.0
     (Scaled Velocity = Velocity * 100.0) to bring the input feature values 
     to a standard statistical variance range. This is crucial for gradient 
     flow and network convergence.
   - SEQUENCE ALIGNMENT: The sequence is padded with zeros or truncated to 
     fit a fixed sequence length of exactly 29 frames (flattened to a shape 
     of 29 steps x 99 features, since 33 keypoints x 3 coordinates = 99).

4. PYTORCH LSTM INFERENCE:
   - The aligned tensor is fed into the trained PyTorch LSTM model.
   - The 2-layer LSTM network processes the 29-step velocity sequence 
     sequentially to capture temporal changes in speed and direction.
   - The final hidden state of the LSTM is passed through a dense layer, 
     ReLU activation, and a Dropout layer to output raw logits.
   - A Softmax activation is applied to the logits to produce class-wise 
     probabilities (0% to 100%) for all 9 motion classes.
   - The class with the highest probability is selected as the prediction.

5. SIDEBAR DASHBOARD VISUALIZATION:
   - The MediaPipe skeleton lines are drawn over the original frame.
   - The frame is padded on the right with a 320-pixel dark panel.
   - The sidebar displays:
     * Active Motion Badge (color-coded to the predicted class)
     * Class Confidence horizontal progress bars for all 9 classes
     * Pose prediction (Standing, Sitting, Crouching, Lying)
     * Real-time Frames Per Second (FPS) metric
   - The final canvas is displayed on screen and written to the output file.


------------------------------------------------------------------------
SECTION 2: LSTM MODEL SPECIFICATIONS
------------------------------------------------------------------------
The model is a custom PyTorch Recurrent Neural Network (RNN) using LSTM cells.

MODEL ARCHITECTURE:
- Input Size: 99 features (33 landmarks x 3 coordinates [x, y, z] per frame)
- Sequence Length: 29 time steps
- Recurrent Layers: 2 stacked LSTM layers
- Hidden Dimension: 64 features per LSTM cell
- Dropout Rate: 0.3 (applied between LSTM layers and in classification block)
- Fully Connected Block:
    - Linear Layer (64 input features -> 32 output features)
    - ReLU Activation
    - Dropout Layer (0.3)
    - Output Linear Layer (32 input features -> 9 output classes)

TRAINING HYPERPARAMETERS:
- Loss Function: Cross-Entropy Loss (nn.CrossEntropyLoss)
- Optimizer: Adam (optim.Adam)
- Initial Learning Rate: 0.001
- Weight Decay: 1e-5
- Learning Rate Scheduler: ReduceLROnPlateau (drops LR by 50% on loss plateau)
- Epochs: 50
- Evaluation Metrics: 100% Accuracy, Precision, Recall, and F1-score on dataset.
- Best Checkpoint Path: model_train/models/motion_lstm_best.pth

OUTPUT CLASSES:
- Class 0: Stationary
- Class 1: Approaching
- Class 2: Backing Away
- Class 3: Fast Across
- Class 4: Slow Approach
- Class 5: Fast Toward
- Class 6: Approach + Stop
- Class 7: Minimal
- Class 8: Across
